# 车道线二分类感知模块 — 设计文档

---

**模型**: ResNet18 (10.6M) | **数据**: TuSimple 6408 帧 | **测试准确率**: 91.56% | **F1**: 94.33% | **lane_out 召回率**: 84.0%  
**框架**: PyTorch 2.x + ONNX Runtime | **日期**: 2026-05

---

## 1. 项目概述

本模块是自动驾驶感知系统的一部分，负责判断当前车辆是否位于自车道（ego-lane）内。输入前视摄像头 ROI 图像，输出 lane_in / lane_out 二分类结果。

### 核心判定规则

在图像底部（y=690）处，以图像中心（x=640）为界，将标注车道线分为左右两侧候选。分别取最靠近中心的一条作为自车道左/右边界。若左右边界均存在且图像中心位于两者之间 → `lane_in`；否则 → `lane_out`。

### 输入输出

| 项目 | 说明 |
|------|------|
| 输入图像 | 1280×720 前视图 → 640×360 ROI → 224×224 |
| 输出 | 1 个 logit → sigmoid → [0, 1] |
| 分类阈值 | 0.5 |
| 推理延迟 | ~2ms (GPU, batch=1) |

## 2. 数据流水线

方案经过三次迭代，最终采用"单ROI 自然标注"（方案A）：每张图仅裁剪一个 640×360 的车辆前方区域，标签完全由 TuSimple 人工标注自动推导，不引入人工虚构负样本。

## 4. 训练策略 (v2 优化)

| 策略 | 配置 |
|------|------|
| 损失函数 | Focal Loss (α=0.25, γ=2.0) |
| 采样 | WeightedRandomSampler (~50:50/batch) |
| 优化器 | AdamW (lr=1e-3, wd=1e-4) |
| 调度器 | CosineAnnealingWarmRestarts (T_0=10) |
| 精度 | AMP 混合精度 (CUDA) |
| 梯度裁剪 | max_norm=1.0 |
| Batch Size | 128 |
| 早停 | patience=10 (基于 val_loss) |

### Focal Loss 原理

\[
\text{FL}(p_t) = -\alpha_t \cdot (1-p_t)^\gamma \cdot \log(p_t)
\]

- 当样本易分时，\((1-p_t)^\gamma \to 0\)，该样本梯度贡献被削弱
- 当样本难分时，\((1-p_t)^\gamma \to 1\)，保持原始梯度

这使模型自动聚焦于 lane_out 等难分样本，是 lane_out 召回率从 47.6% 提升至 84.0% 的核心因素。

## 6. ONNX 部署

支持 PyTorch → ONNX 导出，使用 ONNX Runtime 进行跨平台推理。

- ONNX 模型: `lane_binary.onnx` (43MB)
- 输入: `float32 [1, 3, 224, 224]`
- 输出: `float32 [1, 1]` (logit)
- 精度: PyTorch vs ONNX Runtime 差异 < 1e-7 (浮点误差级)

## 8. 总结

本实验验证了将车道线标注转化为二分类标签、用 ResNet18 学习"车辆是否在自车道内"这一感知决策的可行性。在 TuSimple 全量数据上，测试 F1 达 94.33%，lane_out 召回率达 84.0%，达到工程部署水平。

### 改进方向

- **多高度判定**: 在 y=300/500/690 多处判断，减少局部遮挡误判
- **时序建模**: 输入连续 3-5 帧，利用帧间连续性平滑判决
- **知识蒸馏**: ResNet18 → MobileNetV2，保持精度同时降低延迟
- **多数据集**: 引入 CULane/BDD100K 提升复杂场景泛化能力
- **部署量化**: ONNX INT8 + TensorRT，模型压缩至 ~11MB
